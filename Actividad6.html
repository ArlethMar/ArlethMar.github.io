Siglas

OSI

Este es un modelo que describe cómo se comunican los sistemas de red. Este modelo se usa para establecer reglas y estándares para la infraestructura de red y las aplicaciones.

IP

IP significa “Protocolo de internet”, es un número compuesto que identifica a un dispositivo conectado a internet estás permiten cambiar información entre dispositivos, distinguir entre uno y otro conectado a internet, en resume dirige todos los datos a su destino

TCP

TCP por sus siglas significa “Transmission Control de Transmisión”, que traducido al español es “Protocolo de control de transmisión”. Este es un estándar de comunicaciones que permite la transmisión de datos entre dispositivos por medio de una red

UDP

Es un protocolo del nivel de transporte (encapsulado entre la capa de red y la capa de aplicación del modelo OSI) basado en la transmisión sin conexión de datagramas y representa una alternativa al protocolo TCP. Este a diferencia del concepto anterior puede funcionar sin conexión y es mas rápido, sin embargo no garantiza la entrega de datos.

SSH

Se trata de un método para enviar comandos de forma segura a un ordenador mediante una red no segura, este protocolo usa criptografía para autenticar y encriptar las conexiones entre dispositivos. Permite a los usuarios controlar y modificar sus servidores remotos a través de internet por medio de un mecanismo de autenticación.

FTP

Las siglas de FTP significan File Transfer Protocol, que se traduce como Protocolo de Transferencia de Archivos. FTP es un protocolo que se utiliza para transferir todo tipo de archivos entre equipos conectados a una red, por ejemplo internet. El FTP funciona abriendo dos conexiones que vinculan las computadoras que intentan comunicarse entre si.

SMTP

Son las siglas de “Simple Mail Transfer Protocol” que en español significa protocolo simple de transferencia de correo permite a los ordenadores y servidores intercambiar datos independientemente de su hardware o software.

POP

El protocolo POP, siglas de Post Office Protocol (Protocolo de Oficina Postal),sirve para sincronizar el correo electrónico de Gmail con cualquier cliente de correo compatible, como Outlook, Thunderbird o Apple Mail

DHCP

El protocolo DHCP (Dynamic Host Configuration Protocol) es un conjunto de reglas que asigna direcciones IP y otros parámetros de configuración a los dispositivos conectados a una red. permite a un máquina cliente de la red obtener direcciones IP y otros parámetros de configuración del servidor.

DNS

El protocolo DNS (Sistema de nombres de dominio) es un protocolo de red que traduce nombres de dominio a direcciones IP. Es como la guía telefónica de Internet; simplifica el proceso de búsqueda de sitios web específicos a través de navegadores sitio web.

HTTP

HTTP, de sus siglas en inglés: "Hypertext Transfer Protocol", es el nombre de un protocolo el cual nos permite realizar una petición de datos y recursos, como pueden ser documentos HTML. Se utiliza para transmitir documentos hipermedia, como HTML, imágenes, videos, y otros archivos multimedia.

ARP

El protocolo ARP se encarga de traducir las direcciones IP a direcciones MAC (direcciones físicas). Para realizar esta conversión, el nivel de enlace utiliza las tablas ARP, cada interfaz tiene tanto una dirección IP como una dirección física MAC

MAC

Es un método que permite a varios dispositivos compartir una red de medios. Se encarga de controlar el acceso a los recursos de la capa física de la red, como las bandas de frecuencia, los intervalos de tiempo y los niveles de potencia.

LAN

Los protocolos de red de área local (LAN) son un conjunto de reglas que permiten a los dispositivos comunicarse entre sí en una red local. Los protocolos LAN utilizan métodos como el acceso múltiple con detección de colisión y portadora (CSMA/CD) o el paso de token.
